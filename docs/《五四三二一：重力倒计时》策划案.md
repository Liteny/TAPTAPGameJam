## 核心机制：五级重力系统

**核心思路：** 玩家跳跃初速度恒定(549 px/s)，仅重力加速度变化，实现 5 种跳跃高度。

| 等级 | 名称 | 跳高(瓦片) | 重力(px/s²) | 腾空时间 | 手感 |
|---|---|---|---|---|---|
| **5** | 超轻 | 5 格 | 942 | 1.17s | 月面漂浮 |
| **4** | 轻 | 4 格 | 1177 | 0.93s | 轻盈 |
| **3** | 正常 | 3 格 | 1570 | 0.70s | 标准 |
| **2** | 重 | 2 格 | 2355 | 0.47s | 沉重短促 |
| **1** | 超重 | 1 格 | 4709 | 0.23s | 几乎离地 |

**关键规则：**
- 跳跃高度固定（非蓄力），保证谜题可预测性
- 空中掠过开关不触发，必须站在开关上
- 水平移动速度恒定 200 px/s，不受重力影响
- 开关可无限重复使用

---

## 重力开关设计

- 外观：六边形地板嵌板，中央浮空全息数字
- 5 种颜色：青(5) → 绿(4) → 黄(3) → 橙(2) → 红(1)
- 激活瞬间：数字闪白 → 粒子爆发 → 颜色切换
- 未激活时数字缓慢脉冲(2s 周期)

## 谜题设计原则

1. **可达性锁链：** 开关 A 在重力 X 可达区 → 获得新重力 Y → 解锁开关 B 所在区域
2. **可见预兆：** 出口/关键开关提前可见但够不到 → 驱动解谜欲
3. **杜绝软锁：** 所有开关可重复使用，总有可返回路径
4. **正反双向：** 高重力跳得高但无法钻低缝，低重力反之

### 范例谜题（关卡 3-1）："高不成低不就"

```
布局概要：
- 起点(重力3) → 左侧[5]开关在2格高平台 → 踩后变重力5
- 右侧通道天花板2.5格 → 重力5跳5格会撞头 → 必须用低重力
- 通道深处有[2]开关 → 踩后变重力2 → 通过通道
- 通道出口右侧有[5]开关 → 踩后恢复重力5 → 跳上出口高台
```

## 操作手感参数

| 参数 | 值 |
|---|---|
| 地面速度 | 200 px/s |
| 空中速度 | 150 px/s (75%) |
| 地面加速度 | 2000 px/s² |
| 土狼时间 | 0.08s (5帧) |
| 跳跃缓冲 | 0.12s (7帧) |
| 重力对称性 | 上升=下落 |

输入：A/D 移动，空格/W/↑ 跳跃，R 重置关卡

## 叙事框架

**故事：** 空间站"天梯七号"重力核心失控，维护机器人"小七"必须在倒计时归零前穿越五个重力失常舱段，手动重启核心。

| 倒计时 | 舱段 | 重力 | 情绪 |
|---|---|---|---|
| 5 | 观测穹顶 | 超轻 | 失重、自由 |
| 4 | 生态环廊 | 轻 | 轻盈、生长 |
| 3 | 核心枢纽 | 正常 | 最后的秩序 |
| 2 | 能源舱段 | 重 | 沉重、压迫 |
| 1 | 反应堆底层 | 超重 | 绝境、决意 |
| 0 | — | — | 结局 |

叙事方式：关卡开头/结尾 2-3 行文字弹窗，无对话系统，无 NPC。

```
gravity_countdown/
├── scenes/
│   ├── levels/      (level_1_1.tscn ~ level_6_1.tscn)
│   ├── player/      (player.tscn)
│   ├── switches/    (gravity_switch.tscn)
│   ├── ui/          (hud.tscn, main_menu.tscn, pause_menu.tscn, narra_overlay.tscn)
│   └── props/       (exit_door.tscn, checkpoint.tscn)
├── scripts/
│   ├── autoload/    (gravity_manager.gd, level_manager.gd, save_manager.gd)
│   ├── player/      (player.gd)
│   ├── switches/    (gravity_switch.gd)
│   └── ui/          (hud.gd, main_menu.gd, etc.)
├── assets/
│   ├── sprites/     (player/, tiles/, switches/, ui/, background/)
│   ├── audio/       (sfx/, music/)
│   └── fonts/
└── project.godot
```

**3 个 Autoload 单例：** GravityManager, LevelManager, SaveManager

**核心信号流：**
- `GravityManager.gravity_changed` → Player(物理), HUD(显示), Camera(光晕)
- `LevelManager.level_loaded` → NarraOverlay(文字), SaveManager(进度)
- `ExitDoor.level_complete` → LevelManager(加载下一关)

## 关键风险

| 风险 | 缓解 |
|---|---|
| AI 出图风格不一致 | 统一 prompt 后缀；预留 30min 手动调色 |
| 关卡难度失衡 | Day 1 白盒完成后找人试玩 |
| 物理手感不佳 | 前 5h 锁定手感数值 |
| 重力1高速下落穿透薄平台 | 关键平台 ≥2 瓦片厚；move_and_slide 半步积分 |
| 范围蔓延 | 白盒后不加新机制；严格遵守排期 |
| 单人疲劳 | 每天 ≥5h 睡眠；接受"未完成"，保核心循环 |

## 验证方式

1. **Day 1 验证：** 白盒 6 关全部可通关（自己逐关跑通）
2. **Day 2 验证：** 找 1-2 人试玩，观察卡关点
3. **提交前验证：** Windows .exe 在干净机器可运行；Web 版在 Chrome/Firefox 可运行
4. **核心标准：** 即使美术/音效不完美，6 关必须全部可通关
